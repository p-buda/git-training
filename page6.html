Git & Merging Branches

Before you merge

Before you merge your branches, move to the branch that is your main one (or the one you want the changes to be merged to.) In our case, it will be master.

         git checkout master

Merging

Now, we are ready to merge the branches.

         git merge branch_name

Solving merge conflicts

When you try to merge two branches that have the same file but each has a different code in the same line, you will get a merge conflict. In that case, you have to tell Git which version you want to keep. This is how you can solve the merging conflicts.

Open the file that has a conflict. The conflicting line versions are added into the same file on your current branch and you can decide which lines of the code to keep and which to delete.

Once you are done editing, you have to add and commit the changes, explaining your choice in solving the conflict.

         git add file_name
         git commit -m 'Message about your choice in the merge conflict.'

After that, the branches are merged and they both point to the HEAD of the main branch.

Deleting branches

After your additional branch was merged with the main branch of your project, you can delete it if you no longer need it. To do that, use this command:

         git branch -d branch_name

Note that Git will not allow you to delete a branch that was not merged. But you can force it with the upper case "d," like so: git branch -D branch_name. See more on that topic at Git Branching - Branch Management.
