Git & Making Branches

What is a branch?

When you initialize a Git repository and start making commits, they are all part of one version of your project. Without creating any other versions, you only have one default branch in your project, called by convention master.

A branch is a pointer to the latest commit in a version of your project. When your project has many branches, it has many pointers. Git keeps track of which branch you are currently using by yet another pointer, called HEAD.

To see which commit the branch called "master" points to, use this command:

         git show master

To see the list of all the branches in your project, use this command:

         git branch

The branch that is currently selected will have a star symbol by its name, for example, * master.

Creating branches

Git allows you to branch-off from the main version of your project and start a different version. For example, to add a feature or fix a bug while keeping the main branch intact and available for other programmers to work with till you are ready to merge your changes into it. To do that, you have to create another branch and give it a name.

To create a new branch, type 

            git branch branch_name

Now, to switch to that new branch and put it in your working directory, use:

            git checkout branch_name

Switching between branches

You can switch between all the branches in your project using git checkout branch_name command where "branch_name" is the name of the branch you want to switch to.

But note that when you switch after several commits, you are “back” in your project version to the state your files were when you did the latest commit in the branch you are switching to.
