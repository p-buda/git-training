Git & Fetching & Pulling from Remotes

Fetching changes from the remote repository to the local repository

When you work on the project that you store on GitHub, and it happens that you uploaded the most current version of your code to GitHub from another computer, you can bring the code on your current machine up-to-date by fetching and then pulling the changes from that remote on GitHub.

Another reason to fetch and pull is when you work on the project with multiple users who share the same GitHub repository. You can get the history of all the new files and code they committed to that repository by running the fetch command, like so:

         git fetch

Note that this command is not going to update your code or merge those files to your version of the repository. It will just update your local .git tracking folder.

Updating local repository with the files and changes from the remote repo

Now that we fetched the info from the remote repo about the changes in the project, we can pull those changes to our local version of the project. To do that, we use this command:

         git pull

Note that you can skip the git fetch command and go straight to git pull to update the local files.
