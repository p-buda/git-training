Git & Remote

Adding your code from your local repository to a remote repository

GitHub is that remote location for you (you can just call it a remote).

Command for adding a remote repository's "bookmark" to your local repository

Note that we are not downloading any code yet with this command! It is just like a bookmark.

         git remote add origin LOCATION OF THE REMOTE REPOSITORY

Know that origin is the arbitrary name of that remote location and that's the name we see used most often by default.

Command for pushing your code to the remote repository

Now that you have the remote repository "bookmarked" in your local repo, you can push the code from the local to the remote. Here is how:

         git push origin master

Know that master is the name of our local repository's main branch.

If we push to the remote repository a lot, we can create a shortcut version of the previous command by typing this:

         git push -u origin master

After we press Enter, the next time we want to push from master to the origin, we can type only

         git push

How to check your bookmarks, that is, how to know what remotes you are connected to?

Type this command to see the list of all your remotes.

         git remote -v
