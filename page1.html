Git Basics

What is Git?

Git is an open-source program that enables version control in your projects. Git helps you track changes in your project files as the project progresses.

How does Git keep track of the changes? When you work on a project, for example, you write a program, and you just added a new feature or fixed a bug, you can tell Git to save the current version of your project.

Later you can always revert to any of the saved versions of your project if needed.

Git also allows you to create multiple branches (that is, versions) of your project that can be merged later. That is very useful when you work on a project with a team of developers. Each developer can focus on one feature and then all these features can be merged into the main project (in Git, called master branch).

What is GitHub?

You have a general idea what Git version control program is. So, what is GitHub then? GitHub is one of the cloud-based storage systems that integrate with Git. GitHub allows us to store the project files from our local Git repositories on a remote server so we can easily retrieve them and, most importantly, collaborate with other developers on projects.

The free version of GitHub allows you to store your project files in a way that is public, that is, anyone can see your source code and anyone can download it. That is why you never want to upload to GitHub passwords or any other personal information. Since January 2019 you can also choose to create free private repositories. See more at this article.

There is a great benefit of the public repositories. Organizations like NASA or Microsoft and individuals publish their source code on GitHub in this way and hope that some other developers will download their code and add to it something great that will improve their product or find a bug that would go unnoticed otherwise. That's how open-source projects are being built!

GitHub also has an enterprise version that allows organizations to use it in their private space where nobody from the outside has access to their code. You might use GitHub in this way in your future job.
